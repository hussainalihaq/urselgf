# Ameer Global: What To Do Now

This is the shortest practical setup guide for the current website.

## 1. Truth about the keepalive

The keepalive should **reduce** the chance of Supabase Free pausing.

It does **not** guarantee it forever.

Why:

- Supabase does not officially promise that keepalive traffic is a production-safe anti-pause solution.
- Vercel Hobby cron runs only once per day and is not exact to the minute.
- If cron, Vercel, or Supabase fails for several days, the project can still pause.

Practical conclusion:

- For now, this is the cheapest workable setup.
- For later, the proper fix is `Supabase Pro` or moving the database.

## 2. Vercel values you must add

Open:

- Vercel Dashboard
- Select your Ameer Global project
- `Settings` -> `Environment Variables`
- Add these for `Production`

### Values you create yourself

These are not generated by Vercel. You make them yourself.

- `CRON_SECRET`
- `KEEPALIVE_KEY`
- `ADMIN_LOGIN_CODE`
- `ADMIN_SESSION_SECRET`

Use long random strings for all of them.

Examples:

```text
CRON_SECRET=ameer-cron-2026-very-long-random-string
KEEPALIVE_KEY=ameer-keepalive-2026-very-long-random-string
ADMIN_LOGIN_CODE=your-private-admin-passcode
ADMIN_SESSION_SECRET=ameer-admin-session-2026-very-long-random-string
```

### Values you copy from Supabase

From Supabase:

- `SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`

Where to get them:

1. Open Supabase Dashboard
2. Open your project
3. Go to `Project Settings` -> `API`
4. Copy:
   - Project URL -> use for both `SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_URL`
   - anon key -> `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - service_role key -> `SUPABASE_SERVICE_ROLE_KEY`

### Values you copy from Stripe

From Stripe:

- `STRIPE_SECRET_KEY`
- `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`
- `STRIPE_WEBHOOK_SECRET`
- `STRIPE_PRICE_SINDHRI`
- `STRIPE_PRICE_ANWAR_RATOL`

## 3. Stripe email: two different options

There are **two** kinds of emails:

### Option A: Stripe sends customer receipt emails itself

This does **not** require your own email credentials.

This is the easiest option for customer receipts.

How it works:

1. Stripe Checkout collects the customer email
2. Your checkout code already sends `customer_email` to Stripe
3. Stripe can send the receipt from Stripe

What you must do in Stripe:

1. Open Stripe Dashboard
2. Go to `Settings`
3. Open `Customer emails`
4. Turn on:
   - `Successful payments`
5. Optionally set branding:
   - logo
   - brand color
   - support/public business info

Important:

- Stripe’s automatic receipts are mainly for the **customer**
- They are not your custom admin notification system
- In `test mode`, automatic email receipts are limited and may not send normally to arbitrary test emails

## 4. Custom order emails from your own business

This is what your current code already supports.

It uses `Resend`, not Stripe, to send:

- customer confirmation email
- admin new-order email

Current code path:

- webhook finalizes paid order
- then `api/_lib/email.js` sends email through Resend

To make this work, you must configure Resend.

### Resend setup

1. Create account at [Resend](https://resend.com)
2. Verify a domain or sender email
3. Add these Vercel env vars:

```text
RESEND_API_KEY=...
ORDER_EMAIL_FROM=orders@yourdomain.com
ORDER_EMAIL_ADMIN_TO=your@email.com
```

4. Make sure `STRIPE_WEBHOOK_SECRET` is also set

Without these:

- Stripe can still process payments
- but your custom order emails will not send

## 5. What Stripe cannot do by itself

Stripe can send its own customer receipt email.

Stripe does **not** replace:

- your custom branded order confirmation email flow
- your admin notification email flow

For those, you need an email provider such as:

- Resend
- Postmark
- SendGrid

Current code is already built for `Resend`.

## 6. What to set first, in order

Do these in this order:

1. Fix Supabase envs in Vercel
2. Add `CRON_SECRET`
3. Add `KEEPALIVE_KEY`
4. Add `ADMIN_LOGIN_CODE`
5. Add `ADMIN_SESSION_SECRET`
6. Fix Stripe webhook secret
7. Turn on Stripe customer receipts
8. Configure Resend if you want custom order/admin emails
9. Redeploy on Vercel

## 7. Minimum setup if you have no money right now

If you want the cheapest setup possible today:

1. Keep Vercel Hobby
2. Keep Supabase Free
3. Use the keepalive
4. Turn on Stripe customer receipts
5. Do **not** set up Resend yet unless you want custom emails immediately

That gives you:

- website stays online
- customer can get Stripe receipts
- no new email service bill yet
